Operating cost slabs — payment gateway & AWS

Illustrative three-level model for a six-product cosmetic D2C website (responsive web + admin + checkout). Maple AI Technologies prepared this for planning; it is not a quote from Razorpay or AWS.

Payment gateway charge (this document): 2% of successful card/UPI/netbanking transaction value processed in the month (your monthly GMV through the gateway).
Monthly PG fee (₹) ≈ 0.02 × monthly GMV (₹). Example: GMV ₹10,00,000 → ₹20,000 at 2%.

Peak concurrent shoppers means how many customers are actively browsing or checking out at the same busy moment (not total customers per day). It drives how much server capacity you reserve. Peak API requests is a rough order-of-magnitude for your website + checkout APIs at that same peak (health checks and static assets excluded).

Illustrative GMV uses average order value (AOV) of ₹450 (cosmetic bundles); replace with your forecast: GMV ≈ orders × your AOV, then PG @ 2% = 0.02 × GMV.

Important notes
  1. Payment gateway: Real-world fees depend on the provider (e.g. Razorpay), payment method (UPI vs card vs international), and GST on the gateway fee. The 2% figure here is a planning slab you asked for—not a Razorpay rate card.
  2. AWS: Bands assume a typical stack (app + API + managed database + object storage + CDN in ap-south-1), modest DR/backups, and no heavy ML/media pipelines. Spikes, logs, extra environments, or third-party SaaS are extra.
  3. Cash on delivery (COD): Not included in the 2% card—there is usually no PG MDR on COD; operational cost is fulfilment/returns.
  4. Validate both lines with your actual gateway agreement and an AWS Cost Calculator / solution architect before signing budgets.
